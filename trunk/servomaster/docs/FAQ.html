ServoMaster: FAQ

What is a project status?
Architecture is clear, design is done, Java implementation for two controllers (FT639) and PhidgetServo) is done, the production stability is reached (one of the controllers manages my home HVAC system - uptime is measured in weeks and the interruptions occur only when I upgrade the software for the rest of the system). More are coming.

Why Java?
First of all, this is my current language of choice (though I'm getting back to C++, for memory footprint and performance reasons). Second, it is the best choice for rapid prototyping. Third, and probably the most important, despite all its drawbacks, it is really platform independent. This last one matters a lot, at least to me.

What about C?
No C, because this project is not about implementation, it's about reusable architecture. C++ port, however, is being implemented, with the same architecture (browse the CVS repository). Don't ask about the timeframe. If you want to help, you're welcome.

Are there other hardware controller drivers coming?
You bet. Currently, USB PhidgetAdvancedServo driver is in the works. The next one is probably Ohmark Digital Servo Controller, unless I get to implement my crazy ideas sooner. Why Ohmark? It supports the transitions in hardware.

How do you select the hardware controllers to implement the drivers for?
First one I bought. The rest were donated. If you want the driver for your controller, consider donating one - by now, the implementation is pretty straightforward, it takes a day or two of straight coding.

I've bought the FT639 and put it together, and it doesn't work, what the hell?
The picture at the FerretTronics FAQ page is ambiguous. It looks like a back of DB-9 female connector, while in reality it is a male connector. The signal goes to pin 3, the ground goes to pin 5.

If this fails, check if the signal level on the chip input is high enough. I had to short the 20K resistor to make it work with my Asus P-2B onboard serial controller (well, that was before I sorted out the confusion with the pinout, I'm surprised it worked at all... Hope I didn't bust my serial port...)

What is this "silent mode" thing?
If you are familiar with the servos, you probably know that they can make a sound that can make you jump, especially caught off guard. The application this project is serving is intended to have the servos operate next to people whose comfort level is paramount, so they must not be scared do death periodically when the servos come to life.

As a side effect, this reduces the energy consumption (the servo not getting the pulse goes to a powerdown and consumes about 8mA - thanks again, Chester) and servo wear. Since the servos may be working 24x7, this may be important as well.

Why do I keep getting noClassDefFoundError: javax/comm/UnsupportedCommOperationException?
Because you didn't install the JavaCOMM properly. If you keep getting probems even though you've followed the instructions to the letter, try to go with the JRE instead of JDK - copy the comm.jar to ${JRE_HOME}/lib/ext directory.

As a rule of thumb, the newer the version of the JDK/JRE, the less problems you will have.

Every time the USB device is disconnected and then reconnected again, I get the SecurityException. How to avoid this?
You have to properly mount the usbfs. It would be a good idea to add the following entry to your /etc/fstab:
usbdevfs /proc/bus/usb usbdevfs devmode=0666 0 0
This will enable all the users to access the usbfs. Alternatives are possible, look up the usbfs documentation for that.
