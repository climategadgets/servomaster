ServoMaster: Quick Start

Prerequisites

  • Jakarta Ant, --with-ANT configure option. Minimum version required is 1.4.
  • Sun's javac.jar or tools.jar, --with-JAVA_COMPILER configure option. Examine the Ant documentation to find out where it fits.
  • Apache Xerces-J, --with-XERCES configure option. Current version will suffice.
  • For Linux boxes, Trent Jarvi's RxTx - Java Communications API implementation.

Environment

The ServoMaster is being developed on the Linux and Solaris boxes, and I am not interested in building for Windows. However, it is possible to build it on Windows using Cygwin, and of course it is possible to use it on Windows as soon as you get the jar file (jar distribution, along with source distribution, is planned).

Build

Hardcore OpenSource people will find the build process comfortable - it's simple ./configure && make all install. However, it is highly recommended to run ./configure --help first, because even though the autoconf is used to adjust the build to the system, the tools used (Ant and Xerces) do not have a standard location, thus it has to be specified.

To avoid specifying configure options again and again, it is recommended to use the ./configure-options file to specify them, one item per line. This is how mine looks: 

--with-ANT=/opt/java/jakarta/jakarta-ant-1.3/dist/lib/ant.jar
--with-JAVA_COMPILER=/opt/java/jakarta/jakarta-tools/javac.jar
--with-XERCES=/opt/java/jakarta/jakarta-ant/proposal/myrmidon/lib/xerces.jar

Running the result

There is a class that serves as a testbed and a demo for the package - org.freehold.servomaster.view.Console, and $prefix/bin/ft_view shell script is a wrapper for it. It takes two parameters: the hardware driver class name, and the driver-specific port name. It is completely hardware independent.

There is no dummy class implementing nonexistent controller that doesn't require any hardware, however, the FT639 controller the driver for which has been already implemented doesn't provide any feedback, so it can be used as a demo. The only requirement is that you specify a valid unoccupied serial port ID, like this: 

        $prefix/bin/ft_view \
                org.freehold.servomaster.device.impl.ft.FT639ServoController \
                /dev/ttyS1
where $prefix is where you actually install the package.

That's it, you should be seeing the console right now.
