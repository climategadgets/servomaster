Disconnected Operation
It is often necessary or desirable to keep the abstraction of the servo controller active even if the actual hardware controller is temporarily unavailable. One frequently encountered cause of unavailability, which is important enough by itself, is the USB device being unplugged.

In order to cope with this problem, the disconnected mode was introduced. If the disconnect is explicitly allowed (controller/allow_disconnect feature), then an error when communicating with the hardware bumps the driver into the disconnected mode. The error is never reported to the caller, however, the notification is sent to the DisconnectListeners of the controller so they are aware of the disconnect and can take preventive measures.

The disconnectable device drivers fall into three major categories:

  1. Ones that don't care. A typical example of such a device is FT639 - it is a write-only box, and you can't detect whether it is connected or not. Writing into the port that has a dead controller or no controller at all doesn't cause IOException. Consequently, no special measures must be taken to make this device disconnectable. The controller/allow_disconnect feature is set to true and is immutable.

  2. Dedicated drivers. If a driver class knows how to detect a specific type of hardware controller, the metadata (or, should I say, sufficient part of) for that controller is known beforehand, and it is possible to support the disconnected operation even if the device was never present.

  3. Factory drivers. If a driver class knows how to handle multiple types of hardware controllers, it must detect the required instance of a controller on the startup, otherwise it is not possible to do anything other than wait for a specific instance to arrive and then figure out what it is, because no information about the type of the hardware device can be passed to the controller factory when it is being instantiated.
Needless to say, if you are designing a fail-safe application, class 2 drivers is what you need. As of the moment of writing, ServoMaster offers none, but the class 2 drivers are being worked on for the Phidgets family (currently, all the family is supported by a single class 3 driver).
