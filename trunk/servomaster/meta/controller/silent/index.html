ServoMaster: Metadata: controller/silent

controller/silent feature is true if the hardware is able to shut off the servos.
controller/silent property contains the currently set shutoff timeout.

Servo shutoff is useful for applications where long periods of inactivity may happen - the servo wear and energy consumption are greatly reduced. An example may be found here.

Please note that some controllers (like FT639) are able to support the shutoff on per-controller basis (all the servos are shut off at the same time), some (like PhidgetServo) are able to support the shutoff on per-servo basis (see servo/silent, and some are not able to do it at all.

Use the servo/silent feature and property to set the shutoff timeout for the controllers that support per-servo control. For those, the value set with controller/silent will merely be a default.
